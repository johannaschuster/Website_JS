Kurzanleitung für Javascript Intl API

JavaScript Intl API ist ein Objekt in JavaScript für die Internationalisierung von Webanwendungen.

Internationalisierung, auch bekannt als i18n, bezieht sich auf den Prozess der Anpassung von Software an verschiedene Sprachen, Kulturen und Länder ohne den Quellcode ändern zu müssen.

JavaScript Intl bietet Entwicklern eine einfache und effektive Möglichkeit, Anwendung in den Standards der Region abhängig von den Präferenzen des Nutzers anzuzeigen. Dadurch wird eine bessere User Experience ermöglicht.

JavaScript Intl API in unterschiedlichen Laufzeitumgebungen

Die Intl-API wird sowohl in Webbrowsern als auch in Node.js unterstützt.

JS Intl in Browsern

Die Intl-API wird in allen modernen Webbrowsern unterstützt, einschließlich Google Chrome, Mozilla Firefox, Apple Safari und Microsoft Edge. Die Verwendung der Intl-API in Browsern ist einfach und erfordert keine zusätzlichen Pakete oder Bibliotheken. Die API ist als integriertes Objekt im Browser verfügbar und sofort einsatzbereit.

Anmerkung: Einige ältere Browser unterstützen die Intl-API möglicherweise nicht. Falls Sie solche Browser auch unterstützen müssen, sollten Sie die Verwendung von Polyfills in Betracht ziehen. Intl im Knoten In Node.js wird die Intl-API in neueren Versionen unterstützt. Die Verwendung der Intl-API in Node ist einfach wie in Browsern und erfordert keine zusätzlichen Pakete oder Bibliotheken.

Wozu braucht es diese WEB-API ?

Die JavaScript Intl-API bietet eine Vielzahl von Funktionen und Objekten, die Entwickler verwenden können, um die Internationalisierung von Webanwendungen zu erleichtern.

Das am häufigsten verwendete Objekt ist das Intl.DateTimeFormat-Objekt, das Entwicklern die Möglichkeit gibt, Datums- und Uhrzeitangaben in verschiedenen Formaten anzuzeigen. So zum Beispiel das Anzeigen von Tagen vor Monaten in einigen Ländern oder das Anzeigen von Stunden im 24-Stunden-Format.

Die Funktion DateTimeFormat kann verwendet werden, um ein Datum in das gewünschte Format zu bringen.

Hier ein Beispiel: 


        constdate = newDate('2023-03-21'); 
        constformatter = newIntl.DateTimeFormat('de-DE', { 
          weekday: 'long', 
          year: 'numeric', 
          month: 'long', 
          day: 'numeric', 
        }); 
        console.log(formatter.format(date)); 
        // Ausgabe: "Dienstag, 21. März 2023"

In diesem Beispiel wird das Datum "2023-03-21" für den Wochentag, den Monat und das Jahr im deutschen Sprachraum formatiert. Der DateTimeFormat-Konstruktor akzeptiert ebenfalls verschiedene Optionen, um das Ausgabeformat anzupassen.

Ein weiteres nützliches Objekt in JavaScript Intl ist das Intl.NumberFormat-Objekt, mit dem Entwickler Zahlen in verschiedenen Formaten anzeigen können. Hierzu gehört beispielsweise die Anzeige von Währungsbeträgen, Prozentzahlen oder auch Ganzzahlen.

Hier ein Beispiel:


          const number = 123456.789;
          console.log(
            new Intl.NumberFormat("de-DE", { style: "currency", currency: "EUR" 
          }).format(
            number,
            ),
          );
          // Ausgabe: "123.456,79 €"

In dem Beispielcode wird eine Zahl (number) im Währungsformat für Deutschland formatiert. Dazu wird ein neues Intl.NumberFormat-Objekt erstellt, dem die gewünschte Sprache ('de-DE') und das gewünschte Format (style: 'currency', currency: 'EUR') übergeben werden. Das Objekt wird dann verwendet, um die Zahl zu formatieren und das Ergebnis auf der Konsole auszugeben.

Darüber hinaus bietet JavaScript Intl auch Unterstützung für Sprach-Tagging, eine Methode zur Identifizierung der Sprache, in der Text geschrieben ist. Die Sprache wird dabei durch einen eindeutigen Code identifiziert, der aus einer Sprach- und Regionalkennung besteht.

Hier ein Beispiel:


        constmessages = { 
          en: { 
            greeting: 'Hello, world!', 
          }, 
          de: { greeting: 'Hallo, Welt!', }, 
        }; 
        constlanguage = navigator.language; 
        consttranslatedMessage = messages[language].greeting; 
        console.log(translatedMessage); 
        // Ausgabe: "Hallo, Welt!" (für den Sprachraum "de")

In diesem Beispiel wird die Variable language verwendet, um die bevorzugte Sprache des Nutzers zu ermitteln. Dann wird der entsprechende Text aus dem messages-Objekt abgerufen und angezeigt.
