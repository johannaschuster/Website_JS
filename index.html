Kurzanleitung für Javascript Intl API

JavaScript Intl API ist ein Objekt in JavaScript für die Internationalisierung von Webanwendungen.

Internationalisierung, auch bekannt als i18n , bezieht sich auf den Prozess der Anpassung von Software an verschiedene Sprachen, Kulturen und Länder ohne den Quellcode ändern zu müssen.

JavaScript Intl bietet Entwicklern eine einfache und effektive Möglichkeit, Anwendung in den Standards der Region abhängig von den Präferenzen des Nutzers anzuzeigen. Dadurch wird eine bessere User Experience ermöglicht.

JavaScript Intl API in unterschiedlichen Laufzeitumgebungen

Die Intl-API wird in allen modernen Webbrowsern unterstützt, einschließlich Google Chrome, Mozilla Firefox, Apple Safari und Microsoft Edge. Die Verwendung der Intl-API in Browsern ist einfach und erfordert keine zusätzlichen Pakete oder Bibliotheken. Die API ist als integriertes Objekt im Browser verfügbar und sofort einsatzbereit.

In Node.js wird die Intl-API in neueren Versionen unterstützt. Die Verwendung der Intl-API in Node ist einfach wie in Browsern und erfordert keine zusätzlichen Pakete oder Bibliotheken.

Wozu braucht es diese WEB-API ?

Die JavaScript Intl-API bietet eine Vielzahl von Methoden und Objekten, die Entwickler verwenden können, um die Internationalisierung von Webanwendungen zu erleichtern. So ist es mit wenig Code möglich die Webanwendung für die Standards bestimmer Regionen zu konfigurieren.

Anwendungsbeispiele

Ein häufig verwendetes Objekt ist das Intl.DateTimeFormat-Objekt , das Entwicklern die Möglichkeit gibt, Datums- und Uhrzeitangaben in verschiedenen Formaten anzuzeigen. So zum Beispiel das Anzeigen der Tage vor dem Monat in einigen Ländern oder das Anzeigen von Stunden im 24-Stunden-Format.

Hier ein Beispiel: 


      const options = {
        weekday: 'long',
        year: 'numeric',
        month: 'long',
        day: 'numeric'
      };
      
      const date = new Date();
      
      const formatter = new Intl.DateTimeFormat('de-DE', options);
      const formattedDate = formatter.format(date);
      
      console.log(formattedDate); 
      //Ausgabe: Derzeitiges Datum (z.B. Montag, 27. März 2023)

In diesem Beispiel wird das Intl.DateTimeFormat-Objekt verwendet, um das derzeitige Datum in einem bestimmten Format anzuzeigen. Das Objekt wird mit dem gewünschten Sprach- und Regionencode (hier "de-DE" für Deutsch in Deutschland) und den Formatierungsoptionen (hier "Wochentag", "Jahr", "Monat" und "Tag") initialisiert.

Ein weiteres nützliches Objekt in JavaScript Intl ist das Intl.NumberFormat-Objekt , mit dem Entwickler Zahlen in verschiedenen Formaten anzeigen können. Hierzu gehört beispielsweise die Anzeige von Währungsbeträgen, Prozentzahlen oder auch Ganzzahlen.

Hier ein Beispiel:


          const number = 123456.789;
          console.log(
            new Intl.NumberFormat("de-DE", { style: "currency", currency: "EUR" 
          }).format(
            number,
            ),
          );
          // Ausgabe: "123.456,79 €"

In dem Beispielcode wird eine Zahl (number) im Währungsformat für Deutschland formatiert. Dazu wird ein neues Intl.NumberFormat-Objekt erstellt, dem die gewünschte Sprache ('de-DE') und das gewünschte Format (style: 'currency', currency: 'EUR') übergeben werden. Das Objekt wird dann verwendet, um die Zahl zu formatieren und das Ergebnis auf der Konsole auszugeben.

Mit dem Intl.RelativeTimeFormat-Objekt kann man Zeitangaben in verschiedenen Formaten bzw. Sprachen wiedergeben.

Hier ein Beispiel:


      const relativeTimeFormat = new Intl.RelativeTimeFormat("en-US");
      relativeTimeFormat.format(10, 'seconds');
      //Ausgabe: in 10 seconds
      relativeTimeFormat.format(-10, 'seconds');
      //Ausgabe: 10 seconds ago
      relativeTimeFormat.format(-5, 'month');
      //Ausgabe: 5 months ago
      relativeTimeFormat.format(3, 'year');
      //Ausgabe: in 3 years

Das Beispiel zeigt die Ausgabe der Zeitangaben im englischen Format. Im Code muss hierfür allerdings nur die Anzahl der Tage, Jahre, etc. und die Einheit dazu einem neuen RelativeTimeFormat-Objekt zugewiesen werden. Daraus wird dann ein Satz in der jeweiligen Sprache im richtigen Format gebaut.

Ein weiteres Objekt wird für die verschiedenen Standards von Aufzählungen benötigt. Das Intl.ListFormat-Objekt kann alle Arten von Aufzählungen für die einzelnen Regionen generieren. Das heißt es werden Wörter wie "und" oder "oder" zwischen die einzelnen Wörter gesetzt.

Hier ein Beispiel:


        const listFormat = new Intl.ListFormat("de-DE");
        listFormat.format(['Dafe', 'Daneil', "Gbolahan", "Kelani", "David"]);
        //Ausgabe: Dafe, Daneil, Gbolahan, Kelani und David
        
        const listFormatOr = new Intl.ListFormat("de-DE", {type: 'disjunction'});
        listFormatOr.format(["Bohnen", "Reis", "Salat"])     
        //Ausgabe: Bohnen, Reis oder Salat

Beim ersten Beispiel wurde eine Aufzählung im deutschen Format mit Trennwort "und" generiert. Beim zweiten Beispiel wurde das "und" durch ein "oder" ersetzt. Dies wurde ganz einfach mit der Ergänzung {type: 'disjunction'} erreicht.

Sortierungen von Strings innerhalb eines Arrays kann man mit dem Intl.Collator-Objekt an die jeweiligen Standards anpassen. Bei besonderen Buchstaben, die nicht in allen Sprachen verwendet werden, wie zum Beispiel dem schwedischen "Å", ist die Sortierung von Region zu Region unterschiedlich. Wenn man nun also Wörter in eine Reihenfolge bringen will und ein Wort mit einem solchen Buchstaben beginnt, kann es Regionsabhängig an verschiedenen Plätzen stehen.

Hier ein Beispiel:


        let names = ['Adam', 'Bob','Åslaug', 'Ørjan'];

        let enCollator = new Intl.Collator('en-US', { 
          sensitivity: 'base' 
        });
        console.log(names.sort(enCollator.compare)); 
        //Ausgabe: 'Adam', 'Åslaug', 'Bob', 'Ørjan'
        
        let svCollator = new Intl.Collator('sv-SE', { 
          sensitivity: 'base' 
        });
        console.log(names.sort(svCollator.compare)); 
        //Ausgabe: 'Adam', 'Bob', 'Åslaug', 'Ørjan'

Das Beispiel zeigt genau den oben genannten Fall. Nach englischen Standards erfolgt eine andere Reihung der Wörter, als bei den schwedischen Standards 'Åslaug' steht im schwedischen erst eine Stelle weiter hinten als im englischen Standard.

locales argument

Mit Hilfe des locales argument ist es möglich die genutzte Region des Users festzustellen und die Darstellung einzelner Daten dann individuell anzupassen.

Hier ein Beispiel:


      console.log(new Intl.NumberFormat(locale).format(1234567890));
      //Ausgabe für "en-US": "1,234,567,890"

      //Ausgabe für "de-DE": "1.234.567.890"

Die Ausgabe ist so also individuell an den User angepasst.
